# 開發與部署路線圖
本專案採用分階段的敏捷開發方法，旨在快速驗證核心價值、降低風險，並逐步交付功能完善的產品。
---

## ✅ 第一階段：快速原型設計與可行性驗證 (已完成)
**目標**: 以最低成本驗證核心 AI 功能的技術可行性，確保專案方向正確。
- [x] **提示工程 (Prompt Engineering)**:
  - [x] 使用 Google AI Studio 收集真實巡檢圖像樣本。
  - [x] 反覆測試並優化 AI 模型提示範本，確保能穩定生成準確的 JSON 輸出。
- [x] **核心 AI 流程驗證**:
  - [x] 在前端原型中直接呼叫 Gemini API，驗證端對端的 AI 分析流程。
- [x] **UI 概念驗證**:
  - [x] 以 React 建立一個功能完整的互動式網頁原型。
  - [x] 向專案關係人展示核心使用者流程（拍照 -> AI 分析 -> 數據顯示）。
- [x] **完成階段性目標**: 成功證明 AI 能夠準確地從圖像中提取所需的結構化數據，並獲得專案關係人對 UI 流程的初步認可。
---
## ⏳ 第二階段：最小可行性產品 (MVP) 開發與試點 (進行中)
**目標**: 在第一階段原型的基礎上，建構並部署一個包含完整後端和行動應用程式的核心功能系統版本，供小規模試點使用者進行真實場景測試。
- [x] **雲端基礎設施建置**:
  - [x] 按照 `arch.md` 文件，在 Google Cloud Platform 上建立完整的專案。
  - [x] 設定 Supabase, Google Cloud Storage, Cloud Run 和 Vertex AI 的權限與服務。
- [x] **後端開發**:
  - [x] 在 Cloud Run 上部署所有後端微服務 (auth, task, upload, processing) 的初始版本。
  - [ ] 完成後端 API 的開發與測試。
- [ ] **前端開發**:
  - [ ] 開發一個功能性的 Flutter 行動應用程式。
  - [ ] 實現 `ui.md` 中定義的核心工作流程（登入、任務列表、拍照、審查與儲存）。
  - [ ] 將 Flutter App 與新的後端服務整合。
- [ ] **試點計畫 (Pilot Program)**:
  - [ ] 將 MVP 部署給 5-10 名真實的巡檢員使用。
  - [ ] 密切收集他們在使用過程中遇到的問題和反饋。
- [ ] **階段性目標**: 擁有一個可以在真實環境中運作的端對端系統，並根據試點反饋制定下一階段的優化方向。
---
## 🚀 第三階段：企業級功能增強與擴展 (長期規劃)
**目標**: 在 MVP 的基礎上，增加企業級所需的可擴展性、安全性、可維護性和分析功能，並逐步擴大部署範圍。
- [ ] **監控與維運 (MLOps & DevOps)**:
  - [ ] 為所有雲端服務建立全面的監控儀表板 (Monitoring) 和日誌記錄 (Logging)。
  - [ ] 建立 CI/CD 管線，實現程式碼的自動化測試與部署。
- [ ] **進階功能開發**:
  - [ ] 實現自動化的摘要報告生成 (PDF 格式)。
  - [ ] 開發基於角色的存取控制 (RBAC) 功能。
- [ ] **數據分析與洞察**:
  - [ ] 使用 Looker Studio 或類似工具，連接到 Supabase (PostgreSQL)，建立分析儀表板。
  - [ ] 追蹤巡檢趨勢、設備異常率和 AI 模型性能。
- [ ] **持續優化**:
  - [ ] 根據使用者反饋和數據分析，持續迭代行動 App 的 UI/UX。
  - [ ] 優化後端服務的性能與成本。
