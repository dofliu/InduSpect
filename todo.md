開發與部署路線圖
本專案採用分階段的敏捷開發方法，旨在快速驗證核心價值、降低風險，並逐步交付功能完善的產品。

第一階段：快速原型設計與可行性驗證 (已完成)
目標: 以最低成本驗證核心 AI 功能的技術可行性，確保專案方向正確。

主要活動:

提示工程 (Prompt Engineering):

工具: Google AI Studio。

交付成果: 收集 20-30 張真實的巡檢圖像樣本，反覆測試並優化 AI模型整合規格 中定義的提示範本，確保能夠穩定生成準確的 JSON 輸出。

後端 API 測試 (目前以前端直接呼叫 Gemini API 實現):

工具: Python 腳本 (或目前前端直接呼叫)。

交付成果: 使用從 AI Studio 取得的程式碼，開發一個簡單的 Python 腳本，直接呼叫 Vertex AI API，驗證端對端的 AI 分析流程。目前原型已在前端直接實現此驗證。

UI 概念驗證:

工具: Streamlit 或 Figma (目前以 React 網頁原型實現)。

交付成果: 建立一個簡單的互動式網頁原型或 UI 線框圖，向專案關係人展示核心的使用者流程（拍照 -> AI 分析 -> 數據顯示）。目前已完成一個功能完整的 React 網頁原型。

此階段結束時的目標: 證明 AI 能夠準確地從圖像中提取所需的結構化數據，並獲得專案關係人對 UI 流程的初步認可。此目標已透過目前的網頁原型達成。

第二階段：最小可行性產品 (MVP) 開發與試點 (3-4 個月)
目標: 在第一階段原型的基礎上，建構並部署一個包含完整後端和行動應用程式的核心功能系統版本，供小規模試點使用者進行真實場景測試。

主要活動:

雲端基礎設施建置:

交付成果: 按照 系統架構設計 文件，在 Google Cloud Platform 上建立完整的專案，包括設定 Firebase、Cloud Storage、Cloud Run、Firestore 和 Vertex AI 的權限與服務。

後端開發:

交付成果: 在 Cloud Run 上部署所有後端微服務 (auth, task, upload, processing) 的初始版本，並完成 API 的開發與測試。

前端開發:

交付成果: 開發一個功能性的 Flutter 行動應用程式，實現 UI/UX設計規格 中定義的核心工作流程（登入、任務列表、拍照、審查與儲存），並與新的後端服務整合。

試點計畫 (Pilot Program):

交付成果: 將 MVP 部署給 5-10 名真實的巡檢員使用，密切收集他們在使用過程中遇到的問題和反饋。

此階段結束時的目標: 擁有一個可以在真實環境中運作的端對端系統，並根據試點反饋制定下一階段的優化方向。

第三階段：企業級功能增強與擴展 (持續進行)
目標: 在 MVP 的基礎上，增加企業級所需的可擴展性、安全性、可維護性和分析功能，並逐步擴大部署範圍。

主要活動:

監控與維運 (MLOps & DevOps):

交付成果: 為所有雲端服務建立全面的監控儀表板 (Monitoring) 和日誌記錄 (Logging)。建立 CI/CD 管線，實現程式碼的自動化測試與部署。

進階功能開發:

交付成果: 實現自動化的摘要報告生成 (PDF 格式)。開發基於角色的存取控制 (RBAC) 功能。

數據分析與洞察:

交付成果: 使用 Looker Studio 或類似工具，連接到 Firestore/BigQuery，建立分析儀表板，用於追蹤巡檢趨勢、設備異常率和 AI 模型性能。

持續優化:

交付成果: 根據使用者反饋和數據分析，持續迭代行動 App 的 UI/UX，並優化後端服務的性能與成本。

此階段為持續的過程: 系統將根據業務需求的發展，不斷增加新功能並進行維護。
