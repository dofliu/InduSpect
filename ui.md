使用者介面與體驗 (UI/UX) 設計規格
本文件旨在為 Flutter 開發人員和 UI 設計師提供行動應用程式介面的設計指南和使用者流程。設計應遵循「行動優先」和「人在迴路中」的原則。

1. 使用者流程圖
[登入] --> [選擇巡檢工作] --> [查看檢查表] --> [執行單點巡檢] --> [審查與覆寫] --> [返回檢查表] --> [提交工作] --> [登出]

   |              |                |                  +-----------------+                |                  |
   |              |                +-----------------------------------------------------+                  | (所有點完成後)
   |              | (選擇另一工作)                                                                            |
   +--------------+------------------------------------------------------------------------------------------+

2. 核心介面描述
2.1 登入畫面 (Login Screen)
元件:

App Logo。

電子郵件輸入框。

密碼輸入框。

「登入」按鈕。

「使用 Google 登入」按鈕。

邏輯: 透過 Firebase Authentication 進行身份驗證。

2.2 巡檢工作列表 (Job List Screen)
元件:

顯示使用者名稱和歡迎訊息的標頭。

一個可滾動的列表，顯示所有分配給該使用者的巡檢工作。

每個列表項應顯示：

工作標題。

狀態標籤 (例如：待處理 - 灰色，進行中 - 藍色，已完成 - 綠色)。

工作進度指示器 (例如："5/12 個檢查點已完成")。

互動: 點擊列表項進入「檢查表畫面」。

2.3 檢查表畫面 (Checklist Screen)
元件:

顯示當前工作標題的標頭。

一個包含所有巡檢點的清單。

每個清單項應顯示：

巡檢點名稱和位置。

一個清晰的狀態指示器：

待巡檢 (空心圓圈)。

已巡檢 - 正常 (綠色勾)。

已巡檢 - 異常 (紅色驚嘆號)。

互動:

點擊「待巡檢」的項目，打開相機進行巡檢。

點擊「已巡檢」的項目，進入「審查與覆寫畫面」。

當所有項目都完成後，顯示一個「提交完整報告」的按鈕。

2.4 巡檢畫面 (Inspection Screen - Camera & Processing)
元件:

全螢幕相機預覽。

拍照按鈕。

閃光燈開關。

返回按鈕。

流程:

使用者拍照。

(可選) 設備端進行快速的模糊或亮度檢查，若不合格則提示重拍。

顯示一個覆蓋層 (overlay) 或加載指示器，並提示「圖像上傳與分析中，請稍候...」。

分析完成後，自動跳轉到「審查與覆寫畫面」。

2.5 審查與覆寫畫面 (Review & Override Screen)
這是「人在迴路中」的核心介面，設計必須清晰直觀。

元件:

顯示所拍攝的圖像預覽。

一個表單區域，顯示由 AI 填充的數據：

設備類型: (可編輯的文字欄位)。

讀數 - 數值: (可編輯的數字欄位)。

讀數 - 單位: (可編輯的文字欄位)。

狀況評估: (可編輯的多行文字區域)。

是否存在異常: (一個切換開關 Toggle Switch)。

「確認並儲存」按鈕。

「重拍照片」按鈕。

邏輯: 使用者可以在此畫面驗證 AI 的結果。如果進行了任何修改，inspector_override 標記應設為 true。點擊「確認」後，數據將被寫入 Firestore，並返回到「檢查表畫面」。
